## 🎯 核心特性

### 1. 严格导师系统

**不再是友好的AI助手，而是严格的导师**：

#### 对比示例

**❌ 旧版（简单AI）**
```
用户: "我想做Facebook广告"
AI: "好的！Facebook广告成本约$50-100/用户..."
```

**✅ 新版（严格导师）**
```
用户: "我想做Facebook广告"

导师: "等等。在投广告之前，请先回答我三个问题：

问题1: 你的目标CAC是多少？如何计算出来的？
[必须显示计算过程]

问题2: 你的LTV是多少？CAC/LTV比例是多少？
[必须给出数字和公式]

问题3: 你测试过落地页转化率吗？
如果转化率只有1%，$10000广告费能带来多少付费用户？
[必须计算]

只有回答完这三个问题，我们才讨论是否投广告。

记住：增长黑客的第一原则是什么？
[提示: 漏桶理论]"
```

### 2. 五维度严格评分

**不再是简单的对错，而是多维度评估**：

| 维度 | 分值 | 评估内容 |
|------|------|----------|
| **数据准确性** | 30分 | 数字是否正确（容错2%） |
| **逻辑完整性** | 25分 | 推理链条是否完整 |
| **计算准确性** | 20分 | 公式和过程是否正确 |
| **概念理解** | 15分 | 是否真正理解原理 |
| **策略可行性** | 10分 | 方案是否可执行 |

**通关标准**：
- **A+ (90-100分)**: 完美！解锁2个新关卡 + 大师徽章
- **A (80-89分)**: 优秀！解锁1个新关卡 + 高级提示
- **B (70-79分)**: 良好！解锁1个新关卡
- **C (60-69分)**: 及格，但标记为"脆弱知识点"
- **F (<60分)**: ❌ 不及格，必须重做（最多3次机会）

### 3. 渐进式线索系统

**不直接给答案，而是循循善诱**：

```javascript
// 第1次尝试失败
导师: "方向对了，但你忽略了什么？
      思考：这个问题的核心概念是什么？
      关键词：CAC, LTV, unit economics"

// 第2次尝试失败
导师: "让我给你一个框架：
      健康的生意需要：CAC < LTV × 30%

      例如：如果LTV是$100，CAC应该<$30

      现在重新计算你的数字。"

// 第3次尝试失败
导师: "好，我给你详细步骤：
      1. 计算LTV = ARPU × 平均留存月数 × Gross Margin
      2. 确定健康CAC = LTV × 30%
      3. 计算当前策略的预期CAC
      4. 对比判断是否可行

      按这个步骤重新做一遍。"

// 最后一次
导师: "让我直接给你答案并解释原理...
      [详细讲解]

      但请记住这个教训：在增长黑客中...
      你需要掌握这个知识点才能继续。"
```

```
时间限制：10分钟
允许尝试：3次
容错率：±5%

评分标准：
- 计算正确（50分）
- 逻辑完整（30分）

及格线：70分
```

#### 示例：Cohort留存分析（难度：Expert）

### 5. 五层验证机制

**不能蒙混过关**，每个答案都要经过5层检查：

```javascript
Layer 1: 格式检查（20分）
├─ 是否包含计算过程？
├─ 是否有解释说明？
├─ 结构是否完整？
└─ 字数是否达标？

Layer 2: 完整性检查（20分）
├─ 是否回答了所有问题？
├─ 是否考虑了约束条件？
├─ 是否给出了假设？
└─ 是否有缺失元素？

Layer 3: 计算准确性（25分）
├─ 公式是否正确？
├─ 数值是否准确（±5%容错）？
├─ 单位是否一致？
└─ 中间步骤是否显示？

Layer 4: 逻辑一致性（20分）
├─ 推理链条是否完整？
├─ 因果关系是否合理？
├─ 是否有逻辑跳跃？
└─ 结论是否有数据支持？

Layer 5: 可行性检查（15分）
├─ 方案是否满足约束？
├─ 预算分配是否合理？
├─ 时间规划是否现实？
└─ 风险是否考虑？

最终得分 = Σ 各层得分
通过条件：总分 ≥ 70 且 Layer 3 ≥ 15
```

### 8. 动态难度调整

**智能适应每个学生的水平**：

```javascript
如果表现优秀（平均分>85，用时<50%）：
→ 提高难度
  - 时间限制 × 0.8
  - 数据复杂度 +1
  - 计算步骤 +2
  - 约束条件 +1

如果遇到困难（平均分<50，失败率>60%）：
→ 降低难度
  - 时间限制 × 1.2
  - 提供提示 +1
  - 计算步骤 -1
  - 允许尝试 +1

如果表现稳定（50-85分）：
→ 保持难度
  - 继续当前难度
  - 记录为"掌握"
```
