## 💪 核心功能

### 1. 严格导师评分

**5维度评估**（每个答案都从5个角度打分）：
```javascript
1. 数据准确性 (30分) - 数字是否正确
2. 逻辑完整性 (25分) - 推理链条是否完整
3. 计算准确性 (20分) - 公式和过程是否正确
4. 概念理解 (15分) - 是否真正理解
5. 策略可行性 (10分) - 方案是否可执行

总分 ≥ 70 分 → 通过
总分 < 60 分 → 不及格，必须重做
```

### 2. 50个渐进式挑战

**第一章：基础指标计算**
- 关卡1：LTV基础计算 (难度：简单，100分)
- 关卡2：CAC Payback Period (难度：中等，200分)
- 关卡3：Blended CAC (难度：中等，200分)
- 关卡4：Unit Economics (难度：中等，400分)

**第二章：逻辑推理**
- 关卡5：转化率下降之谜 (难度：中等，300分)
- 关卡6：Cohort留存分析 (难度：困难，500分)
- 关卡7：流失率突增 (难度：困难，600分)

**第三章：策略设计**
- 关卡8：6个月增长策略 (难度：困难，800分)
- 关卡9：留存率提升 (难度：困难，700分)
- 关卡10：战略转型 (难度：专家，1000分)

... 共50个关卡

### 3. 渐进式线索系统

**不直接给答案，而是逐步引导**：

```
第1次失败 → 方向提示
"思考：这个问题涉及哪些核心概念？
 关键词：LTV, CAC, unit economics"

第2次失败 → 框架提示
"使用这个框架：
 健康生意需要 CAC < LTV × 30%
 例如：LTV=$100 → CAC应该<$30"

第3次失败 → 详细步骤
"按这些步骤：
 1. 计算LTV = ARPU × 留存月数 × Gross Margin
 2. 确定健康CAC = LTV × 30%
 3. 计算当前CAC
 4. 对比判断"

最后一次 → 完整答案 + 深度讲解
```

### 4. 动态难度调整

**根据表现自动调整**：
- 表现优秀（平均分>85） → 提高难度
- 遇到困难（平均分<50） → 降低难度
- 表现稳定 → 保持难度

**个性化调整**：
- 识别弱点（计算/逻辑/策略）
- 针对性训练推荐
- 题目变体生成

### 5. 真实案例库

**6个深度案例**（框架支持100+）：
1. Dropbox推荐计划 (K=0.65)
2. Facebook "7天10好友"
3. Slack "2000条消息"
4. Airbnb的Craigslist集成
5. Hotmail的PS签名
6. PayPal的推荐奖励

每个案例包含：
- 详细背景
- 完整数据
- 数学分析
- 可应用模板

## 💡 使用技巧

### 如何获得高分

**1. 完整的计算过程**
```
❌ 错误：LTV = $420
✅ 正确：
LTV = ARPU × 留存月数 × Gross Margin
   = $50 × 12 × 0.7
   = $420
```

**2. 清晰的逻辑推理**
```
❌ 错误：所以应该降价
✅ 正确：
因为 CAC ($490) > LTV×30% ($126)
说明获客成本过高
所以应该：
1. 降低CAC（优化渠道）或
2. 提高LTV（提价/提升留存）
```

**3. 考虑约束条件**
```
❌ 错误：月费涨到$200
✅ 正确：
月费从$49涨到$116
理由：
- 在市场可接受范围内
- 仍低于竞品
- 保持价格竞争力
```

**4. 量化的预期**
```
❌ 错误：效果应该不错
✅ 正确：
预期效果：
- CAC降低30%（从$490到$343）
- Payback从14月降到10月
- ROI提升40%
```
