# 📖 书籍转游戏 - AI执行模板

## 🎯 目标
任何人拿到一本书后，按照本模板逐步给AI下指令，即可生成一个完整的"代入式问答游戏"，效果完全复制 growth-hacker-game。

---

## 📋 准备工作

### 第一步：准备材料
```
需要准备：
1. 一本书的电子版（PDF/TXT/Markdown）
2. Claude Code 或其他AI编程助手
3. 一个空文件夹

示例：
- 书名：《XXX实战手册》
- 文件夹：~/my-training-game/
```

### 第二步：提取书籍内容（可选技能）
如果你有书籍内容生成的skill，可以先让AI提取：
- 关键概念（20-30个）
- 真实案例（10-15个）
- 常见错误（15-20个）
- 决策框架（5-10个）

### **阶段3：游戏主逻辑（Day 4，6-8小时）**

#### Prompt 3.1：创建游戏引擎
```
创建 crisis-mission.html，包含完整的游戏逻辑：

**游戏状态管理：**
- timeRemaining（剩余时间）
- intelCollected（已收集情报数）
- actionsCompleted（已完成的行动）
- currentQuestion（当前问题索引）
- correctAnswers（正确答案数）

**核心函数：**
1. initLevel() - 初始化关卡
   - 加载关卡数据
   - 重置游戏状态
   - 初始化资源系统
   - 渲染UI

2. startTimer() - 启动倒计时
   - 每秒更新时间显示
   - 时间<30分钟时变红+闪烁
   - 时间归零时触发失败

3. takeAction(actionId) - 执行行动
   - 检查是否重复
   - 检查资源是否足够
   - 消耗时间和精力
   - 显示反馈弹窗
   - 解锁情报
   - 30%概率触发随机事件

4. readyToAnswer() - 准备回答
   - 检查最少完成2个行动
   - 停止计时
   - 开始对话

5. startDialogue() - 对话系统
   - 逐个显示问题
   - 4选1单选
   - 选择后显示反馈
   - 3.5秒后下一题

6. evaluatePerformance() - 评估表现
   - 计算得分（正确率）
   - 判断通过/失败
   - 显示结果弹窗
   - 解锁下一关

**界面布局：**
- 顶部：任务头部（关卡标题、目标、统计数据）
- 左侧主面板：场景卡片+行动选择
- 右侧边栏：资源面板+情报列表+提示

**弹窗系统：**
- 行动反馈弹窗（带图标+内容+时间消耗）
- 对话弹窗（问题+4选项+反馈）
- 结果弹窗（成功/失败+评价+按钮）
- 事件弹窗（随机事件+选择）

请创建完整的HTML文件，包含所有逻辑和DOM操作。
```
